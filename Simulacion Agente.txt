Hola,

Necesito que generes el proyecto completo de la Fase A siguiendo exactamente estas especificaciones. Esto no es una fase de IA ni optimización, es infraestructura mínima funcional.

Estamos construyendo el núcleo determinista del sistema para talleres mecánicos.

Objetivo de esta fase:

Validar que el taller puede:

Registrar un vehículo en menos de 10 segundos.

Actualizar el estado en menos de 2 segundos.

Consultar estado por matrícula o teléfono.

Generar mensaje determinista basado en datos reales.

No usamos IA generativa en esta fase.

STACK DEFINIDO (OBLIGATORIO)

Backend:

Express

Base de datos:

PostgreSQL

Usar pg o Prisma simple (sin complejidad innecesaria)

Frontend:

JavaScript vanilla

Sin React

Sin frameworks frontend pesados

MODELO DE DATOS

Tabla: vehicles

Campos:

id (uuid, primary key)

plate (string, required, indexed)

phone (string, required)

status (enum exacto):

EN_REVISION

ESPERANDO_PIEZA

PRESUPUESTO_PENDIENTE

LISTO

last_event (string, nullable)

updated_at (timestamp auto update)

active (boolean default true)

No añadir más campos.

ENDPOINTS REQUERIDOS

POST /vehicles

Crear vehículo

Recibe: plate, phone

Estado inicial: EN_REVISION

GET /vehicles?active=true

Listar vehículos activos

PATCH /vehicles/:id/status

Actualizar status

Actualizar updated_at automáticamente

GET /vehicles/by-phone/:phone

Devuelve vehículo activo

Si no existe → { error: "NO_ACTIVE_VEHICLE" }

GET /vehicles/by-plate/:plate

Devuelve estado actual y última actualización

LÓGICA DETERMINISTA
